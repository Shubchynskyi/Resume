    Tic-Tac-Toe

    Ein webbasiertes Tic-Tac-Toe-Spiel mit adaptivem Design für Desktops, Tablets und mobile Geräte. Die Anwendung nutzt eine Progressive Web App (PWA)-Umgebung, um die Installation auf dem Startbildschirm zu ermöglichen. Sie bietet einen Einzelspielermodus mit vier Schwierigkeitsstufen, einen lokalen Zwei-Spieler-Modus und Online-Mehrspieler über WebSocket.

    Implementierte Funktionen

    • Adaptives Design: Responsive Benutzeroberfläche, optimiert für Desktops, Tablets und mobile Geräte, mit PWA-Unterstützung für die Installation auf dem Startbildschirm.
    • Mehrere Spielmodi:
      • Einzelspieler mit vier Schwierigkeitsstufen.
      • Lokaler Zwei-Spieler-Modus für gemeinsames Spielen auf einem Gerät.
      • Online-Mehrspieler über WebSocket mit Sitzungsverfolgung und Inaktivitätstimer.
    • Inaktivitätstimer: Schließt automatisch inaktive Spiele, um Ressourcenverschwendung zu vermeiden.
    • Robuste Tests: Enthält Unit-, Integrations- und Selenium-End-to-End-Tests.

    Verwendete Technologien

    • Backend: Spring Boot und WebSocket-Unterstützung.
    • Frontend: Thymeleaf für dynamisches Rendering, Bulma CSS für ein modernes, responsives Design und JavaScript.
    • Deployment: Containerisiert mit Docker und bereitgestellt über eine Jenkins CI/CD-Pipeline.

    Quests-Webanwendung

    Webanwendung zur Erstellung und Durchführung von Textquests (Spring Boot, JPA).

    Implementierte Funktionen

    • Benutzerverwaltung: Verwaltung von Benutzern mit Rollen, wodurch rollenbasierte Zugriffe und Funktionen für verschiedene Benutzertypen bereitgestellt werden.
    • Quest-Erstellung: Erstellen von Quests aus Textvorlagen.
    • Quest-Bearbeitung: Bearbeiten von Quests auch nach der Erstellung, um Flexibilität und Aktualisierungen zu ermöglichen.
    • Tests: Umfassende Unit-, Integrations- und End-to-End-Tests mit JUnit, Testcontainers und Selenium, um Zuverlässigkeit sicherzustellen und Nutzerszenarien in einer simulierten realen Umgebung zu validieren.
    • Logging: Persistente Protokollierung zur Unterstützung der Überwachung und Fehlerbehebung. Protokolle werden gespeichert und bleiben über Anwendungsaktualisierungen hinweg erhalten.
    • Automatisierte CI/CD-Pipeline: Das Projekt wird mit Jenkins auf einem Server in Docker gebaut, getestet und bereitgestellt, um kontinuierliche Integration und Auslieferung zu gewährleisten. Die Anwendung wird vor der Bereitstellung in isolierten Docker-Containern getestet, um Stabilität zu überprüfen.

    Verwendete Technologien

    • Backend: Java mit Spring Boot, Spring Data JPA, Liquibase für Datenbankmigrationen.
    • Frontend: Thymeleaf für die dynamische Seitendarstellung, kombiniert mit HTML, JavaScript und Bootstrap für responsives Design.
    • Tests: JUnit für Unittests, Testcontainers für Integrationstests und Selenium für End-to-End-Webtests, alles innerhalb von Docker-Umgebungen laufend.
    • Deployment: Die Anwendung wird mit Docker und Docker Compose verpackt und bereitgestellt, verwaltet durch eine Jenkins CI/CD-Pipeline mit sicherer Verwaltung von Anmeldeinformationen.
